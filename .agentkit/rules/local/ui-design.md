# UI/UX Rules (e-КТРМ)

## Design sources and precedence
- Главный визуальный источник: `prototype.html`.
- Подтверждение состава вкладок/полей: `TECH_SPEC.md` + `PDF_ORDERS_DETAILED.md`.
- BPMN jpg используются для проверки логики маршрутов, не для финальной стилизации компонентов.

## Global UI requirements
- Язык интерфейса: русский.
- Desktop-first MVP достаточно.
- Нужно сохранять общий паттерн:
  - header;
  - sidebar/process line;
  - карточки секций;
  - табовая навигация;
  - нижняя панель действий.

## Navigation rules
- Header разделы:
  - `Главная` (disabled/basic shell допустим),
  - `Заявки` (рабочий),
  - `Реестр сертификатов` (рабочий),
  - `Справочники` (disabled или read-only без CRUD).

## Required states
- `default`
- `active`
- `disabled`
- `loading`
- `validation error`
- `readonly autofill`
- `success`
- `archived`

## Order-specific UI constraints
- Ордер 3:
  - wizard по подтвержденным вкладкам: заявитель, адрес, ОПС, схема, продукция, приложение, документы, примечание.
- Ордер 4:
  - read-only/controlled форма сертификата с вкладками заявки, заявителя, ОПС, продукции, приложения.
- Ордер 5:
  - формы приостановления/прекращения и переоформления по подтвержденным вкладкам.

## Forbidden UI behaviors in MVP
- Нельзя реализовывать неподтвержденные вкладки как рабочие экраны:
  - `Блок модератора`,
  - `Ход выполнения`,
  - `Подписи`,
  - другие непроверенные разделы.
- Если такие вкладки нужны для визуального каркаса, они должны быть disabled/hidden без бизнес-логики.

## UX action labels (mandatory examples)
- Для отправки заявки заявителем использовать формулировку: `Подписать и отправить`.
- Для сертификата использовать действие: `Подписать` (mock-sign).
